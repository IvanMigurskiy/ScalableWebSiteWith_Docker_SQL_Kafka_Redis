# Бюджетирование

Проект представляет собой систему для планирования и анализа личных финансов, основанную на концепции бюджетирования.

## Описание функционала

Приложение должно содержать следующие данные:
- планируемый доход
- планируемый расход
- пользователь

### Реализуемый API:

- Создание нового пользователя
- Поиск пользователя по логину
- Поиск пользователя по маске имени и фамилии
- Создание планируемого дохода
- Получение перечня планируемых доходов
- Создание планируемого расхода
- Получение перечня планируемых расходов
- Подсчет динамики бюджета за заданный период

#Краткое описание лабораторных работ

- **Лабораторная 1:** Введение в проектирование архитектуры системы с помощью DSL, создание базовых диаграмм и документации.
- **Лабораторная 2:** Разработка REST API с использованием FastAPI, создание первых сервисов для работы с пользователями и бюджетами.
- **Лабораторная 3:** Расширение API, интеграция с базами данных (PostgreSQL, MongoDB), добавление хранения данных.
- **Лабораторная 4:** Внедрение асинхронных механизмов, кеширование, оптимизация запросов, тестирование системы.
- **Лабораторная 5:** Реализация паттернов CQRS, интеграция с Kafka и Redis для асинхронной обработки.
- **Лабораторная 6:** Контейнеризация всей системы, настройка Docker Compose, автоматизация деплоя и тестирования.

Используемые средства

- Язык программирования: Python 3.11
- Фреймворк: FastAPI
- Базы данных: PostgreSQL, MongoDB
- Асинхронные брокеры: Kafka, Redis
- Контейнеризация: Docker, Docker Compose
- Инструменты разработки: Git, VSCode, Postman (для тестирования API)
- Дополнительные библиотеки: Pydantic, SQLAlchemy, MongoDB, aiokafka, aioredis

# Инструкция по запуску проекта "Бюджетирование" через Docker Compose

Этот проект включает в себя микросервисы `user_service` и `budget_service`, каждый из которых можно запускать отдельно или вместе через Docker Compose.

## Предварительные требования

- Установленный Docker и Docker Compose
  [Инструкция по установке Docker](https://docs.docker.com/get-docker/)

## Запуск системы

1. Перейдите в директорию, где находится файл `docker-compose.yml`. В вашем случае, например, это папка lab2.

2. Выполните команду для сборки и запуска всех сервисов:

```bash
docker compose up -d
```

Это запустит оба сервиса в фоновом режиме.
Доступ к сервисам

API user_service доступен по адресу:http://localhost:8000

API budget_service доступен по адресу:http://localhost:8001


## Документация API

Для ознакомления с доступными эндпоинтами откройте:http://localhost:8000/docs — документация Swagger UI для user_servicehttp://localhost:8001/docs — документация Swagger UI для budget_service

## Остановка системы
Чтобы остановить и удалить все контейнеры:
```bash
docker compose down
```
